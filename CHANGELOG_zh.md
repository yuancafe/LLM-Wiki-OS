# 更新日志

[简体中文] | [English](./CHANGELOG.md)

本项目记录了从一个简单的 CLI 技能进化为全面知识操作系统的全过程。

## [1.0.0] - 2026-05-09 (OS 级重生)

### 新增
- **重大更名**：项目从 `llm-wiki-skill` 正式进化为 **LLM Wiki OS**，强调对知识流的调度能力。
- **通用企业级连接器**：通过 MCP 增加了对企业微信、钉钉、Slack、Discord 和 Google Docs 的支持。
- **3 层调度溯源引擎**：实现了优先级检索逻辑：
  - 第一层：本地物理原始文件 (Physical Raw)。
  - 第二层：结构化中继记忆 (NotebookLM, iMA, 飞书)。
  - 第三层：实时网页抓取 (OpenCLI)。
- **Chrome Dev MCP 回退机制**：针对强反爬网站，使用浏览器可访问性快照进行“透视”抓取。

### 变更
- 重构了 `source-registry.tsv`，预设映射增加至 150+。
- 规范化文档体系，采用英文优先、多语言并行的模式。

## [0.4.0] - 2026-05-09 (自生长里程碑)

### 新增
- **自生长引擎**：引入 `sync_growth` 指令，AI 可自动扫描 Obsidian 库并建议新主题。
- **层次化库架构**：从扁平目录升级为 `根目录/主题/raw` 的树状结构，支持无限扩展。
- **研究方向演进**：AI 根据用户真实产出，动态建议更新 `purpose.md`。

## [0.3.0] - 2026-05-09 (动态适配)

### 新增
- **动态发现协议**：深度缝合 `opencli list`，实现对新网站的零代码自动支持。
- **刷新协议**：新增 `refresh-opencli` 指令，保持适配器实时同步。

## [0.2.0] - 2026-05-09 (多平台强化)

### 新增
- **原生 MCP 支持**：直接集成 B 站和 小红书的专属 MCP 提取器。
- **OpenCLI 桥接**：初步完成对 OpenCLI (v1.7+) 生态的适配。

## [0.1.0] - 2026-05-09 (初始集成)

### 新增
- 基于 `sdyckjq-lab/llm-wiki-skill` 进行二次开发。
- 完成在 Agent Skills 目录的初始部署。
- 配置了首个 Obsidian Vault 连接。
